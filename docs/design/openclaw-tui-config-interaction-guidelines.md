# OpenClaw TUI 配置交互设计规范

## 目的

这份文档沉淀本次 OpenClaw TUI 配置页交互整理后的稳定规则，供后续修改 `工具权限`、`Agents 配置` 以及相邻配置路由时复用。

目标只有两个：

1. 让页面上看到的交互形态和真实行为一致。
2. 把影响范围控制在当前路由内，避免顺手改坏别的编辑流程。

## 适用范围

当前优先适用于以下 OpenClaw TUI 配置路由：

- `工具权限`
- `Agents 配置`

在相邻配置页复用这些模式时，也应优先遵守这里的规则，除非该路由已经有更明确的既有约束。

## 核心原则

### 1. 交互类型要和控件外观一致

- 选择型字段用 picker，不要伪装成行内切换器。
- 文本或数值字段用 popup editor，不要看起来像可直接内联编辑。
- 列表项用明确的“编辑 / 删除 / 新增”语义，不要依赖占位空行或隐式模式切换。

### 2. 自动保存只在路由本地生效

- `工具权限` 和 `Agents 配置` 使用 route-local auto-save。
- picker 内移动选项不保存，只有确认才保存。
- popup editor 输入过程中不保存，只有确认才保存。
- 保存成功保持安静，不弹 success toast。
- 保存失败显示 error toast。

### 3. 不把“清空”藏进空输入提交

- 空提交默认是非破坏性的 no-op。
- 真正的清空动作必须显式触发。
- 对支持清空的行，统一使用 `Delete` 或 `Backspace`。

### 4. 影响范围要收紧

- 改 `工具权限` 和 `Agents 配置` 时，不要顺带改 `环境变量`、`Workspace 文件管理` 或共享编辑器的通用行为。
- 如果只是局部路由需要特殊交互，优先做 route-local 处理，不要扩散到全局编辑工作流。

## 具体交互规则

### Tools 路由

#### 权限档位

- 使用单行展示：`配置档位: <当前值>`。
- 不再在页面正文里常驻显示整条内联选项条。
- `Enter` 打开 picker。
- picker 只展示受支持的档位。
- 当前值如果是非标准保留值，可以继续显示在主页面，但 picker 不应伪造预选状态。

#### 允许列表 / 拒绝列表

- 规则分成独立的 `允许列表` 和 `拒绝列表` 小节。
- 现有规则行按真实列表项渲染。
- `Enter` 编辑当前规则。
- `Delete` 删除当前规则。
- `+ 添加允许规则` / `+ 添加拒绝规则` 作为显式新增入口。
- 新增和编辑都走 popup editor。
- 不允许插入空白占位行来启动编辑。
- 空白确认无效，popup 保持打开，直到输入非空值或用户取消。

### Agents 路由

#### 模型配置

- `默认模型` 使用单行展示：`默认模型: <当前值>`。
- 每个回退模型使用单行展示：`回退模型: <当前值>`。
- `+ 添加回退模型` 保持显式新增入口。
- `Enter` 在默认模型、现有回退模型、添加回退模型这三类行上都走 picker。
- 现有回退模型的 picker 需要排除主模型和其他已选回退模型，但保留当前行自己的值，方便重开后直接取消。
- 不再允许模型行用 `Enter` 做循环切换。

#### 运行参数

- 四个字段统一渲染为 `标签: [值]`：
  - `工作区路径: [<value>]`
  - `超时时间（秒）: [<value>]`
  - `上下文 Token 数: [<value>]`
  - `最大并发数: [<value>]`
- 方括号值区是必需的，它用来强调“这是字段值”，不是普通尾随文本。
- `Enter` 打开 popup editor。
- `Delete` / `Backspace` 显式清空当前字段。

#### 正文排版与可读性

- `Agents 配置` 正文继续保留 `模型配置` 和 `运行参数` 两个区块，不额外插入摘要区或第三个主区块。
- 顶部说明文案使用弱化样式展示，并在第一个区块前保留一行明确的空白间隔，避免说明和字段挤在一起。
- 内层区块边框和标题要弱于外层主面板，不要和外围内容框争抢注意力。
- 字段标签在固定标签列内左对齐，不要为了“排整齐”把标签整体推得太靠右。
- 值列必须跨 `默认模型`、`回退模型`、运行参数行、`+ 添加回退模型` 保持同一条竖向起点。
- 字段行前导留白保持紧凑，当前基线是首字距区块左边框 2 列，只留出焦点强调线和最小呼吸感。
- 选中态不再依赖字面 `>` 前缀。彩色模式使用浅背景加左侧细强调线，`NO_COLOR` 模式继续使用可见的反转强调。
- 不在目录里的模型值继续保留 `value (供应商未配置)` 这种可见文案形态，只通过样式弱化尾随状态，不改写成新的文本结构。

## 显式清空与保留值规则

### 默认模型

- `默认模型` 必须有显式 unset 路径。
- 这条路径不依赖打开 picker。
- 在默认模型行按 `Delete` 或 `Backspace` 时，应通过现有 auto-save 路径清空值。

### 保留的非字符串运行时值

- 如果运行时字段来自旧配置或非标准类型，主页面可以保留展示，避免偷偷吞掉用户数据。
- 但用户必须仍然有办法主动清空它。
- 清空字段时，不只要清空当前可见字符串，还要清掉对应的 preserved seed。
- 清空 `超时时间（秒）` 时，还要一并清掉 route-local legacy-timeout 状态，避免用户已经清空后，旧提示还残留。
- runtime popup 在空字符串或全空白确认时，必须保持 non-destructive。

## 视口与可见性规则

- 当 `工具权限` 或 `Agents 配置` 内容超过可视区域时，当前选中行必须保持可见。
- 这类滚动行为优先使用 derived viewport，不新增持久 scroll state，除非已经证明做不到。
- 重点保证以下行在短终端里仍可见：
  - Tools 的 allow / deny 规则行
  - Agents 的 `默认模型`
  - Agents 的回退模型行
  - Agents 的 `+ 添加回退模型`
  - Agents 的运行时参数行
- 包含说明文字、warning、preserved note 的块可以继续换行，但不能把当前选中交互行挤出视口。

## 可见文案与快捷键规则

- 对启用 route-local auto-save 的 `工具权限` 和 `Agents 配置`，不显示保存按钮。
- 这两个路由也不显示 `Ctrl+S` 保存提示。
- 不要在这两个路由上保留无效的 `Ctrl+S` 交互暗示。
- 相邻仍采用显式保存模式的路由，继续保留它们自己的保存提示，不受这里影响。

## 实现约束

- 优先少写代码、少加抽象、少引入跨文件耦合。
- 如果现有 route-local 逻辑已经能承载新交互，优先沿用，不要为了“统一”而重写共享编辑系统。
- 如果后端行为直接借鉴上游 `cc-switch`，尽量保持行为与命名一致；仓库特有适配逻辑优先旁路拆出。

## 验证要求

每次调整这套交互时，至少要覆盖下面几类验证：

- 焦点交互测试：picker、popup editor、删除、显式清空、新增规则、删除规则。
- 渲染测试：`标签: [值]` 这种确切形态、`Agents 配置` 的标签左对齐和值列对齐、短视口下选中行可见，以及无字面 `>` 前缀时仍有明确焦点强调。
- 邻近回归测试：`环境变量`、`Workspace 文件管理`、共享编辑器脏状态确认等不在本次范围内的行为。
- 格式检查：Rust 代码改动后跑 `cargo fmt --check`。

## 使用方式

- 改 OpenClaw 配置页交互前，先读这份文档，再看对应 spec / plan。
- 如果新需求和这份规范冲突，先在 spec 里明确为什么要偏离，再改实现。
- 如果某条规则已经被新设计替换，更新这份文档，不要让项目级入口挂着过时引用。
